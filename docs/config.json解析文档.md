# 日志配置文件 `config.json` 解析文档

**文档版本**：1.0  
**最后更新**：2026年6月6日  
**适用模块**：`logger.py` + `config_manager.py`（Dream-with-star 项目）

本文档详细解释日志系统配置文件 `config.json` 中每个字段的含义、默认值、取值范围及实际效果。您可以根据项目需求调整这些参数，实现灵活的日志行为。

---

## 字段详解

### 顶层 `log` 对象
所有日志配置的根节点。

---

### 1. `level` — 全局日志级别

| 属性 | 说明 |
|------|------|
| **类型** | 字符串 |
| **可选值** | `"DEBUG"`, `"INFO"`, `"WARNING"`, `"ERROR"`, `"CRITICAL"` |
| **默认值** | `"INFO"` |
| **作用** | 第一道过滤器，只有级别 ≥ 该值的日志才会被处理（无论控制台还是文件）。例如设为 `"INFO"`，则 `DEBUG` 日志完全忽略。 |

**示例**：
```json
"level": "INFO"
```

---

### 2. `console` — 控制台输出配置

#### `console.enabled`
- **类型**：布尔值 (`true`/`false`)
- **默认值**：`true`
- **作用**：是否将日志输出到终端（标准输出）。

#### `console.level`
- **类型**：字符串，可选值同全局 `level`
- **默认值**：`"INFO"`
- **作用**：控制台独立级别阈值，只有 ≥ 该值的日志才打印到屏幕。通常设为 `"INFO"` 避免调试信息干扰。

#### `console.color`
- **类型**：布尔值
- **默认值**：`true`
- **作用**：是否启用彩色输出。不同级别颜色：`DEBUG` 灰色，`INFO` 绿色，`WARNING` 黄色，`ERROR` 红色，`CRITICAL` 紫红加粗。

**示例**：
```json
"console": {
  "enabled": true,
  "level": "INFO",
  "color": true
}
```

---

### 3. `file` — 文件输出配置

#### `file.enabled`
- **类型**：布尔值
- **默认值**：`true`
- **作用**：是否将日志写入文件。关闭后所有文件相关功能（轮转、清理、警告分离等）均失效。

#### `file.level`
- **类型**：字符串
- **默认值**：`"DEBUG"`
- **作用**：文件输出的独立级别阈值。通常设为 `"DEBUG"` 以便记录详细信息到磁盘。

#### `file.path`
- **类型**：字符串（目录路径）
- **默认值**：`"logs"`
- **作用**：日志文件的存储目录。支持相对路径（相对于程序工作目录）或绝对路径。

**示例**：
```json
"file": {
  "enabled": true,
  "level": "DEBUG",
  "path": "logs",
  "rotation": { ... }
}
```

---

### 4. `rotation` — 日志轮转与清理配置

#### `max_size_mb`
- **类型**：整数（MB）
- **默认值**：`100`
- **作用**：单个日志文件的最大大小。当当前文件超过该值且 `by_size` 为 `true` 时，自动轮转（关闭当前文件，创建新文件，序号 +1）。

#### `retention_days`
- **类型**：整数（天）
- **默认值**：`30`
- **作用**：日志文件保留天数。系统启动时自动删除修改时间早于 `当前日期 - retention_days` 的文件。例如设为 `30`，则保留最近 30 天的日志。

#### `max_total_size_mb`
- **类型**：整数（MB）
- **默认值**：`1024`
- **作用**：日志目录总大小上限。当所有日志文件大小之和超过该值时，从最旧的文件开始删除，直到总大小低于阈值。

#### `warning_separate`
- **类型**：布尔值
- **默认值**：`true`
- **作用**：是否将 `WARNING`、`ERROR`、`CRITICAL` 级别的日志额外存储到 `logs/warnings/` 子目录。方便快速定位问题。

#### `by_date`
- **类型**：布尔值
- **默认值**：`true`
- **作用**：是否按日期轮转。每天生成一个新的日期目录/文件名（例如 `20260606-001.log`）。

#### `by_size`
- **类型**：布尔值
- **默认值**：`true`
- **作用**：是否按文件大小轮转。与 `max_size_mb` 配合使用。

**示例**：
```json
"rotation": {
  "max_size_mb": 100,
  "retention_days": 30,
  "max_total_size_mb": 1024,
  "warning_separate": true,
  "by_date": true,
  "by_size": true
}
```

---

### 5. `performance` — 性能调优配置

#### `async_enabled`
- **类型**：布尔值
- **默认值**：`false`
- **作用**：是否启用异步日志写入。启用后，日志调用仅将消息放入内存队列，后台线程负责写文件，减少主线程阻塞。适合高频日志场景。

#### `async_queue_size`
- **类型**：整数（条数）
- **默认值**：`5000`
- **作用**：异步队列的最大容量。当队列满时，新的日志会触发“丢弃最旧一条”的策略，防止内存溢出。

**示例**：
```json
"performance": {
  "async_enabled": true,
  "async_queue_size": 5000
}
```

---

### 6. `console_window` — 独立监控窗口（Windows 专用）

- **类型**：布尔值
- **默认值**：`true`
- **作用**：在 Windows 系统下，自动弹出一个独立的 PowerShell 窗口，实时滚动显示当前日志文件内容（类似 `tail -f`）。方便在不干扰主程序控制台的情况下监控日志。非 Windows 平台自动忽略。

**示例**：
```json
"console_window": true
```

---

### 7. `format` — 输出格式配置

#### `format.file_json`
- **类型**：布尔值
- **默认值**：`false`
- **作用**：控制文件输出格式。
  - `false`：使用 **KV 格式**（空格分隔的 `key=value`），便于人工阅读。
  - `true`：使用 **JSON 格式**，每行一个 JSON 对象，便于机器解析和日志采集系统（如 ELK、Loki）集成。

**示例**：
```json
"format": {
  "file_json": false
}
```

## 实际效果说明（以当前日期 2026年6月6日为例）

假设使用上述配置运行程序：

- **日志存储位置**：`./logs/20260606-001.log`
- **警告及以上日志额外存储**：`./logs/warnings/warning_20260606-001.log`
- **控制台输出**：绿色 `INFO` 级别日志，红色 `ERROR` 等，带有颜色
- **独立窗口**：弹出一个新终端窗口，实时滚动显示 `20260606-001.log` 内容
- **异步模式**：主线程几乎不受日志写入影响，队列容量 5000
- **文件大小限制**：当日志文件超过 100 MB 时，自动创建 `20260606-002.log`
- **自动清理**：保留最近 30 天（即 2026年5月7日及之后的日志），且目录总大小不超过 1 GB

## 注意事项

1. **配置文件路径**：默认从 `data/configs/config.json` 加载（可由 `main.py` 中的 `config.load_file` 指定）。
2. **热重载**：若启用配置监控（`config.start_watch_thread()`），修改配置文件后日志系统会自动应用新设置，无需重启。
3. **字段大小写敏感**：所有字段名必须与文档一致（例如 `"level"` 不能写成 `"Level"`）。
4. **布尔值**：必须使用 `true` / `false`（JSON 标准），不要使用 `True` / `False`。
5. **Windows 颜色**：如果未安装 `colorama`，控制台颜色可能显示为乱码
