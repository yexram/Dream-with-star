# 配置与日志模块接口文档

## 概述

本文档描述了两个核心工具模块：

- **`config_manager`**：全项目通用配置管理器，支持 JSON/YAML 配置文件、环境变量覆盖、热重载和变更监听。
- **`logger`**：多级别异步日志系统，支持控制台彩色输出、文件轮转、多进程安全、独立日志监控窗口及配置热重载。

两个模块均采用单例模式，线程安全，适用于 Python 3.14，支持 Windows 11 。

---

## 模块依赖

| 依赖库 | 版本 | 用途 | 是否必须 |
|--------|------|------|----------|
| `portalocker` | ≥2.7.0 | 多进程文件锁，保证日志序号唯一 | 推荐（无则降级为扫描，有极低概率冲突） |
| `colorama` | ≥0.4.6 | Windows 控制台 ANSI 颜色支持 | 推荐（无则颜色显示乱码） |
| `pyyaml` | ≥6.0 | YAML 配置文件支持 | 仅当使用 YAML 格式时必装 |

# 一、配置管理器 (`config_manager`)

## 1.1 初始化

配置管理器是单例，无需手动实例化。导入后全局唯一实例 `config` 已自动创建。

```python
from src.tools.config_manager import config
```

### 1.1.1 加载配置（三种方式）

#### 方式一：仅从文件加载
```python
config.load_file("path/to/config.json")          # JSON 格式
config.load_file("path/to/config.yaml", format="yaml")   # YAML 格式
```

#### 方式二：链式加载（推荐）
```python
defaults = {"app": {"name": "MyApp", "debug": False}}
config.load_chain(defaults, "config.json", env_prefix="APP_")
```
优先级：`defaults` < 文件内容 < 环境变量（以 `APP_` 开头的变量，例如 `APP_APP_DEBUG=true` 会覆盖 `app.debug`）

#### 方式三：仅从环境变量加载
```python
config.load_from_env(prefix="MYAPP_")
```

### 1.1.2 设置默认值（可选）
```python
config.set_defaults({"log": {"level": "INFO"}})
```

## 1.2 核心 API

### 获取配置值
```python
value = config.get("log.level", default="INFO")
value = config.get("database.host")   # 支持点号路径
```

### 动态修改配置（运行时，不持久化）
```python
config.set("log.level", "DEBUG")
```

### 注册配置变更监听器
```python
def on_config_change(key_path: str, old_value: Any, new_value: Any):
    print(f"{key_path} 从 {old_value} 变更为 {new_value}")

config.watch(on_config_change)
```

### 热重载（手动触发）
```python
config.reload()   # 重新加载配置文件，自动通知监听器
```

### 自动热重载（后台线程监控文件变化）
```python
config.start_watch_thread(interval=1.0)   # 每秒检查一次文件修改时间
config.stop_watch_thread()
```

### 获取完整配置（深拷贝）
```python
all_cfg = config.get_all()
```

## 1.3 功能边界

### ✅ 能做什么
- 加载 JSON / YAML 配置文件（自动识别后缀）
- 环境变量覆盖（支持布尔、整数、浮点数自动转换）
- 嵌套配置通过点号路径访问
- 配置热重载并通知所有监听器
- 线程安全的读写

### ❌ 不能做什么
- 不支持 TOML、INI、XML 等格式（可自行扩展）
- 环境变量映射规则固定（`APP_LOG_LEVEL` → `log.level`），不可自定义分隔符
- 不支持配置加密存储（密码等敏感信息需自行处理）
- 不支持配置回滚（无历史版本管理）

# 二、日志系统 (`logger`)

## 2.1 初始化

日志系统也是单例，推荐使用模块级函数 `log_init_from_config` 从配置管理器加载配置。

### 2.1.1 从配置管理器初始化（推荐）
```python
from src.tools.config_manager import config
from src.tools.logger import log_init_from_config

# 先确保 config 已加载配置文件
config.load_file("data/configs/config.json")
# 初始化日志系统（自动注册热重载回调）
log_init_from_config(config)
```

### 2.1.2 手动初始化
```python
from src.tools.logger import log_init

log_init(
    level="INFO",                      # 全局级别
    console_enabled=True,
    console_level="INFO",
    console_color=True,
    file_enabled=True,
    file_level="DEBUG",
    log_dir="logs",
    async_enabled=False,               # 是否异步写入
    async_queue_size=5000,
    max_size_mb=100,                   # 单个日志文件大小上限（MB）
    retention_days=30,                 # 日志保留天数
    max_total_size_mb=1024,            # 日志目录总大小上限（MB）
    warning_separate=True,             # 是否将 WARNING 及以上级别单独存储到 warnings/ 目录
    rotation_by_date=True,             # 按日期轮转
    rotation_by_size=True,             # 按大小轮转
    console_window=True                # Windows 下打开独立控制台窗口实时显示日志
)
```

### 2.1.3 独立控制台窗口说明
- **仅 Windows + PowerShell 环境有效**（其他平台自动忽略）
- 窗口使用 `Get-Content -Wait` 实时追踪最新日志文件
- 程序退出时自动关闭窗口
- 可通过 `log_open_console()` / `log_close_console()` 手动控制

## 2.2 日志记录 API

### 基本输出
```python
from src.tools.logger import log_debug, log_info, log_warning, log_error, log_critical

log_debug("调试信息")
log_info("用户登录成功", user_id=12345)        # 自动附加自定义字段
log_warning("磁盘空间不足", used_percent=95)
log_error("数据库连接失败", exc_info=True)     # exc_info=True 自动捕获堆栈
log_critical("系统即将崩溃")
```

### 上下文绑定（线程局部）
```python
log_bind(request_id="abc-123", session_id="xyz")
log_info("处理请求")          # 会自动携带 request_id 和 session_id
log_unbind("request_id")      # 解绑单个字段
log_clear_context()           # 清空当前线程所有绑定字段
```

### 动态修改日志级别
```python
log_set_level("DEBUG")        # 全局级别切换
```

### 手动刷新与清理
```python
log_flush()                   # 强制将所有缓冲日志写入磁盘
log_cleanup()                 # 手动执行过期日志清理（通常自动执行）
log_close()                   # 关闭所有文件句柄并停止异步线程
```

## 2.3 配置文件示例

将以下内容保存为 `data/configs/config.json`（与配置管理器配合）：

```json
{
  "log": {
    "level": "INFO",
    "console": {
      "enabled": true,
      "level": "INFO",
      "color": true

    "file": {

      "level": "DEBUG",

        "max_size_mb": 100,

      "async_enabled": true,
      "async_queue_size": 5000

    "format": {
      "file_json": false
    }
  }
}
```

## 2.4 输出格式

### 控制台输出（带颜色）
```
2025-01-15 10:30:45 | 12345 | 67890 | main.py:42 | INFO | 用户登录成功 [user_id=12345]
```

### 文件输出（KV 格式，一行一条）
```
time=2025-01-15 10:30:45 pid=12345 tid=67890 file=main.py line=42 level=INFO msg=用户登录成功 user_id=12345
```

### JSON 格式文件输出（配置 `format.file_json = true`）
```json
{"time": "2025-01-15T10:30:45", "pid": 12345, "tid": 67890, "file": "main.py", "line": 42, "level": "INFO", "msg": "用户登录成功", "user_id": 12345}
```

## 2.5 日志轮转与清理规则

- **按日期**：每天生成一个新文件 `YYYYMMDD-001.log`，跨天自动递增。
- **按大小**：同一日内当前文件超过 `max_size_mb` 时，序号 +1 生成新文件（如 `20250115-002.log`）。
- **多进程安全**：通过扫描目录 + `portalocker` 文件锁确保序号唯一。
- **清理策略**：
  - 删除 `retention_days` 天前的日志文件。
  - 若日志目录总大小超过 `max_total_size_mb`，删除最旧文件直到低于阈值。
- **警告分离**：`WARNING` 及以上级别日志同时写入 `logs/warnings/warning_YYYYMMDD-XXX.log`。

## 2.6 功能边界

### ✅ 能做什么
- 多级别、多目标输出（控制台 + 文件）
- 异步/同步模式切换
- 自动捕获异常堆栈（`exc_info=True`）
- 线程局部上下文（`bind` / `unbind`）
- 配置热重载（日志级别、目录、异步开关等变更实时生效，不丢日志）
- 多进程安全写入
- 自动清理过期日志
- Windows 独立实时监控窗口

### ❌ 不能做什么（限制与注意事项）
- **不支持分布式追踪**：无内置 Trace ID 自动传递，需自行通过 `bind` 实现。
- **不支持日志采样/限流**：高频重复日志会完整输出，可能影响性能。
- **不支持日志发送到远程服务器**（如 Kafka、Elasticsearch）—— 可自行扩展 `LogFileManager`。
- **控制台颜色在 Windows 无 colorama 时无效**（显示原始 ANSI 码，影响可读性）。
- **独立控制台窗口仅限 Windows PowerShell**：Linux/macOS 下该特性自动禁用。
- **异步队列满时默认丢弃最旧日志**（可修改源码改为阻塞模式）。
- **日志格式不可完全自定义**：当前仅支持 KV 和 JSON，不支持自由模板（如有需求可扩展）。

## 三、快速上手示例

```python
# main.py
import sys
from pathlib import Path
sys.path.insert(0, str(Path(__file__).parent))

from src.tools.config_manager import config
from src.tools.logger import log_init_from_config, log_info, log_error

def main():
    # 1. 加载配置
    config.load_file("data/configs/config.json")
    # 2. 启动配置热重载监控（可选）
    config.start_watch_thread(interval=2.0)
    # 3. 初始化日志系统
    log_init_from_config(config)

    # 4. 使用日志
    log_info("应用启动成功", version="1.0.0")
    log_error("模拟错误", error_code=500, exc_info=True)

if __name__ == "__main__":
    main()
```

运行后：
- 控制台显示带颜色的日志。
- 文件写入 `logs/20260606-001.log` 和 `logs/warnings/`。
- 弹出一个独立 PowerShell 窗口实时滚动显示日志内容。

## 四、常见问题（FAQ）

**Q: 独立控制台窗口没有弹出？**  
A: 检查三点：① Windows 系统；② `log.console_window` 为 `true`；③ `log.file.enabled` 为 `true`；④ PowerShell 可用。

**Q: 配置文件修改后日志系统没有自动重载？**  
A: 需要先调用 `config.start_watch_thread()` 启动监控线程。同时确保日志系统已通过 `log_init_from_config` 注册了回调。

**Q: 多进程运行时日志文件序号冲突怎么办？**  
A: 已通过 `portalocker` 和扫描机制尽量避免。若未安装 `portalocker`，冲突概率极低（几毫秒内两个进程同时创建新文件时可能发生），建议安装该库。

**Q: 异步模式下程序退出时部分日志丢失？**  
A: `log_close()` 会等待最多 5 秒让队列清空。若 5 秒内未完成，会打印警告但不会阻塞退出。可以手动调用 `log_flush()` 再退出。

**Q: 如何临时禁用文件日志？**  
A: 调用 `log_init(file_enabled=False)` 或修改配置文件中的 `log.file.enabled` 并触发重载。
